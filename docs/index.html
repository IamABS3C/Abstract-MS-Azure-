Before you deploy · Sentinel only

Create the Entra app first

ARM can't create app registrations, so the Sentinel Destination needs an Entra app you make ahead of time. Five minutes, once.

1 · Register the app

Entra ID → App registrations → New registration (e.g. Abstract-Sentinel-App, single tenant). Save the Application (client) ID and Directory (tenant) ID.

2 · Add a client secret

Certificates & secrets → New client secret. Copy the Value immediately — it's shown once.

3 · Grab the service principal object ID

Entra ID → Enterprise applications → your app → Object ID. This goes in the wizard so the template grants it Monitoring Metrics Publisher + Monitoring Contributor on the DCR.

4 · Deploy

Run the Sentinel Destination template below. It builds the workspace, Sentinel, DCE, custom table, DCR and the role assignments.

5 · Paste into Abstract

Client ID, Client Secret, Tenant ID, plus the deployment's DCR Immutable ID, DCE URL and Stream Name outputs.
